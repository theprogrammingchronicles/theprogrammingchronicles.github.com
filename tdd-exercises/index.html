Test Driven Development Exercises
Step by Step Exercises and Examples to Learn Test Driven Development

The exercises are coded in Java, but I think it would be easy to migrate to other languages. I've tried to use basic JDK elements and avoid to use any particular framework or library, except for the unit testing and mocking libraries.

The unit tests are written with JUnit 4.0 and the automatic mocking with Mockito. Even so there are some exercises that show you how to make unit tests or mocks without any framework or library help such JUnit or Mockito.

The build system is Maven 2 and all the projects generate a lot of useful information with mvn site command.

Organization of the Exercises Projects

The most of the exercises are about the implementation of an application called WikiAddressbook. I have planned to publish information and notes about this imaginary application. In short you can think of it as an application to manage a global address book where anonymous people can create, edit, update and delete contacts. Everyone in the world can contribute to create the content of the bigger address book of the world through the magic of Web 2.0 Collective Intelligence.

The exercises are organized in several lessons and appendix. Every lesson has its own readme file where you can look for the goals of the lesson. Each lesson is divided in several projects that shows individual steps of the incremental and iterative process of TDD. Some steps show very little increments, so they are micro-steps such as the implementation of a very little new behaviour, a very little refactor after getting a green light and so on. Other exercises show how the code look likes after several iterations of the TDD process.

There are some lessons or exercises that do not have anything to do with the WikiAddressbook application. They are isolated exercises showing an important an specific concept of the TDD methodology .
